🎁 Bonus: Deep Dive into logger.js

A complete walkthrough of src/logger.js — how it's structured, how every log request flows through it, and the design decisions behind transport modes, Loki batching, the ring buffer, the singleton pattern, and graceful shutdown.

🏗️ How logger.js Is Structured

The file is 250 lines organized into clear layers. Here's the full anatomy:

LEVELS { debug: 0, info: 1, warn: 2, error: 3 } — numeric priority for filtering
TRANSPORTS ['local', 'direct', 'alloy'] — allowed transport modes
APP_NAME process.env.APP_NAME || package.json name — identifies this app in Loki
APP_VERSION version.json — tracks release version in every log entry
class Logger
  ├─ constructor(options) Sets level, transport, Loki creds, buffer, flush timer, ring buffer
  ├─ Public API
  │  ├─ debug(msg, meta) → _log('debug', ...)
  │  ├─ info(msg, meta) → _log('info', ...)
  │  ├─ warn(msg, meta) → _log('warn', ...)
  │  └─ error(msg, meta) → _log('error', ...)
  ├─ Transport Management
  │  ├─ getTransport() Returns current mode: 'local'|'direct'|'alloy'
  │  ├─ setTransport(mode) Switch at runtime, auto-starts/stops flush timer
  │  └─ getRecentLogs(count) Returns last N entries from ring buffer
  ├─ Loki Connection Test
  │  └─ testLokiConnection() Promise → { ok, status, message }
  ├─ Internal Methods
  │  ├─ _log(level, msg, meta) Build entry, check level, push to ring buffer, call _write()
  │  ├─ _write(level, entry) stdout/stderr + optional Loki buffer add
  │  ├─ _startFlushTimer() setInterval(2s) with .unref()
  │  ├─ _stopFlushTimer() clearInterval
  │  ├─ _flushLoki() Splice buffer, call _pushToLoki(batch)
  │  └─ _pushToLoki(entries) HTTP POST to /loki/api/v1/push with Basic Auth
  └─ destroy() Graceful shutdown: stop timer, flush remaining buffer
module.exports = new Logger() ← Singleton! One instance shared across the whole app

Key Design Decisions

DecisionWhatWhy
Singletonmodule.exports = new Logger()Every file that does require('./logger') gets the same instance. One config, one buffer, one timer — no duplication.
_ prefixAll internal methods start with _Convention to signal "don't call this directly." JavaScript has no private keyword (pre-ES2022), so _ is the standard marker.
Level numbers{ debug: 0, info: 1, warn: 2, error: 3 }Simple numeric comparison: if (LEVELS[level] < this.level) return. Set level to "warn" → only warn (2) and error (3) pass.
Top-level constsAPP_NAME, APP_VERSION outside the classRead once at module load. Never change. Available to the class and any future additions without re-reading files.
Silent failures.catch(() => {}) on Loki pushLogging should NEVER crash the app. If Loki is down, logs are on stdout anyway. The app marches on.

🧠 Core Concepts — Inside logger.js

🔢

Log Levels as Numbers

{ debug:0, info:1, warn:2, error:3 }. The logger compares the incoming level's number against this.level. If the number is too low, the log is silently dropped. This single comparison is the entire filtering mechanism — no if/else chains, no switch statements.

🚛

Transport Modes

local = stdout only (dev). direct = stdout + HTTP push to Loki (quick setup). alloy = stdout only (Alloy agent tails). The mode is read from LOG_TRANSPORT env var at startup, but can be switched at runtime via setTransport() without restart.

📦

Batch Buffer

_lokiBuffer[] collects entries in memory. Flushed when it hits 10 entries OR every 2 seconds. splice(0) atomically empties the array and returns the batch — no race conditions. Reduces HTTP calls from ~60/min to ~6/min.

💍

Ring Buffer (Recent Logs)

_recentLogs[] keeps the last 200 entries. When full, shift() drops the oldest. The admin dashboard reads this via getRecentLogs(50) — no Loki or file reads needed. It's an in-memory circular window.

🔄

The Singleton Pattern

module.exports = new Logger() exports an instance, not the class. Node.js caches modules, so every require('./logger') gets the same object. One config, one buffer, one timer. The constructor runs exactly once — at first import.

⏱️

Timer .unref()

The 2-second flush timer uses setInterval().unref(). Without .unref(), Node.js stays alive forever waiting for the timer. With it, if the app has nothing else to do, the process can exit gracefully — the timer won't hold it hostage.

🔑

HTTP Basic Auth for Loki

Credentials are Buffer.from(user+':'+token).toString('base64'). This is NOT encryption — it's just encoding. HTTPS encrypts the transport layer. Basic Auth is the simplest HTTP auth scheme and is what Grafana Cloud expects.

🕐

Nanosecond Timestamps

Loki expects timestamp strings in nanoseconds. JavaScript gives milliseconds. We append "000000": 1712345678123 → "1712345678123000000". The string format is important — Loki parses it as a 19-digit number.

🛡️

Silent Failure Design

All Loki errors are caught and ignored: .catch(() => {}). If Loki is down, the app continues normally. Logs are already on stdout captured by pm2. This follows the principle: "logging must never break the application."

🗂️

Structured JSON Entries

Every entry is { timestamp, app, version, level, message, meta? }. The meta field is optional — only added when passed. This structure makes logs parseable by Loki | json, greppable by scripts, and human-readable in terminals.

🔌

Graceful Shutdown (destroy)

destroy() stops the timer and flushes the remaining buffer. Call it on process.on('SIGTERM'). Without it, logs buffered in the last 2 seconds are lost on shutdown. pm2 sends SIGINT/SIGTERM before killing.

🌐

Dynamic Protocol (http/https)

The logger auto-detects https: vs http: from the LOKI_URL and picks the right Node.js module. Grafana Cloud always uses HTTPS, but local Loki testing may use HTTP. The URL constructor handles port detection too.

📖 Code Explained — Full logger.js Walkthrough

Click any line to highlight it. Every method explained in detail.

Module Constants (Lines 1–5)

1 const https = require('https'); Node.js built-in HTTPS module. Used for pushing logs to Grafana Cloud Loki (always HTTPS in production).
2 const http = require('http'); HTTP fallback for local Loki instances during development. The logger auto-detects the protocol from LOKI_URL.
3 const LEVELS = { debug: 0, info: 1, warn: 2, error: 3 }; Maps level names to numbers. Higher number = higher severity. if (LEVELS[level] < this.level) return; skips low-priority logs in one comparison.
4 const TRANSPORTS = ['local', 'direct', 'alloy']; Whitelist of valid modes. Used for validation: if (!TRANSPORTS.includes(mode)). Invalid modes fall back to 'local'.
5 const APP_NAME = process.env.APP_NAME || require('../package.json').name; Env var first (for per-deployment names like "pipelinemaster"), then package.json fallback. Evaluated once at module load.
6 const APP_VERSION = require('../public/version.json').version; Reads version from version.json (updated by npm run bump). Appears in every log entry and Loki label. Filter by release in Grafana.

Constructor — Initialization (Lines 7–18)

7 constructor(options = {}) { Accepts an optional config object. In practice, env vars are used. The options are for testing: new Logger({ level: 'debug', transport: 'local' }).
8 this.level = LEVELS[level] !== undefined ? LEVELS[level] : LEVELS.info; If the level string is invalid (e.g., "verbose"), default to "info" (1). The ternary prevents undefined from breaking comparisons.
9 if (!TRANSPORTS.includes(this.transport)) this.transport = 'local'; Invalid transport? Fall back to 'local' (safest mode — no network calls). Never crash on bad config.
10 this.lokiUrl = options.lokiUrl || process.env.LOKI_URL || ''; Grafana Cloud push URL. Empty string is falsy — used for "not configured" checks: if (!this.lokiUrl).
11 this._lokiBuffer = []; // batch: 10 max or every 2s Array that collects log entries before pushing. splice(0) in _flushLoki() atomically empties it.
12 this._lokiMaxBatch = 10; this._lokiFlushMs = 2000; Tuning knobs. 10 entries max per batch, 2-second timer. Trade-off: higher batch = fewer HTTP calls but more latency.
13 if (this.transport === 'direct' && this.lokiUrl) { this._startFlushTimer(); } Only start the timer if transport=direct AND Loki URL is set. No URL = no timer = no wasted resources.
14 this._recentLogs = []; this._maxRecentLogs = 200; Ring buffer for the admin dashboard. Keeps last 200 entries in memory. No disk I/O, no Loki query needed for recent logs.

Public API — The 4 Log Methods (Lines 15–18)

15 debug(message, meta) { this._log('debug', message, meta); } Forwards to _log with level='debug'. Filtered out when LOG_LEVEL is 'info' or higher (default). Use for verbose tracing.
16 info(message, meta) { this._log('info', message, meta); } Standard operational messages: "Server started", "Item created", "Health check OK". The default visible level.
17 warn(message, meta) { this._log('warn', message, meta); } Something unexpected but not broken: slow responses (>1s), invalid IDs, oversized input. Investigate in Grafana.
18 error(message, meta) { this._log('error', message, meta); } Something broke: unhandled exceptions, malformed JSON, 500 errors. Goes to stderr. Alert-worthy in production.

_log() — The Heart of the Logger (Lines 19–26)

19 if (LEVELS[level] < this.level) return; The level gate. If incoming level (e.g., debug=0) is below threshold (e.g., info=1), exit immediately. No entry built, no I/O, no buffer — zero cost for filtered logs.
20 const entry = { timestamp, app, version, level, message }; Build the structured log object. Timestamp is ISO 8601. APP_NAME and APP_VERSION from module constants.
21 if (meta !== undefined) entry.meta = meta; Meta is optional context: { id: 4, name: "Widget" }. Only added when provided. Keeps entries clean when there's nothing extra to say.
22 this._recentLogs.push(entry); Add entry to the ring buffer (for admin dashboard). Always happens, regardless of transport mode.
23 if (this._recentLogs.length > this._maxRecentLogs) this._recentLogs.shift(); If buffer exceeds 200, drop the oldest entry. shift() removes from the front. This is the "ring" in ring buffer.
24 this._write(level, entry); Hand off to _write() which handles stdout/stderr routing and optional Loki buffer injection.

_write() — Output Routing (Lines 25–30)

25 const line = JSON.stringify(entry); Serialize to one-line JSON. Every log entry is a single line — crucial for pm2/Alloy log file tailing.
26 if (level === 'error') process.stderr.write(line + '\n'); Errors go to stderr (fd 2). pm2 captures to *-error.log. Shell redirection: 2>error.log.
27 else process.stdout.write(line + '\n'); Everything else (debug/info/warn) goes to stdout (fd 1). pm2 captures to *-out.log.
28 if (this.transport === 'direct' && this.lokiUrl) { Only buffer for Loki when in direct mode AND URL is configured. Local/alloy modes skip this entirely.
29 this._lokiBuffer.push(entry); Add the entry object (not the JSON string) to the buffer. It gets re-serialized in _pushToLoki() with nanosecond timestamps.
30 if (this._lokiBuffer.length >= this._lokiMaxBatch) this._flushLoki(); If buffer hits 10, flush immediately — don't wait for the timer. Prevents unbounded memory growth during burst logging.

Loki Push — The Network Layer (Lines 31–42)

31 this._lokiFlushInterval = setInterval(() => this._flushLoki(), 2000); The 2-second timer. Even if the buffer has only 1 or 2 entries, they'll be sent within 2 seconds. Maximum latency guarantee.
32 if (this._lokiFlushInterval.unref) this._lokiFlushInterval.unref(); .unref() tells Node.js: "don't keep the process alive for this timer." Without it, node src/server.js would hang forever after Ctrl+C.
33 const batch = this._lokiBuffer.splice(0); splice(0) removes ALL elements and returns them. The buffer is now empty. Atomic: no entry is lost or duplicated between the read and the clear.
34 this._pushToLoki(batch).catch(() => {}); // silently drop Fire and forget. If the push fails, the batch is gone. Logs are already on stdout. Empty catch prevents unhandled rejection crash.
35 const ns = new Date(e.timestamp).getTime() + '000000'; Milliseconds → nanosecond string. 1712345678123 + "000000" = "1712345678123000000". String concatenation, not math.
36 streams: [{ stream: { app, version, environment }, values }] Loki push format: one stream with labels, containing an array of [ns_timestamp, json_line] values. Multiple entries per push.
37 'Authorization': 'Basic ' + Buffer.from(user + ':' + token).toString('base64') HTTP Basic Auth header. 1540346:glc_eyJv... → base64. Not encrypted — HTTPS provides the encryption layer.
38 timeout: 5000, 5-second timeout on the HTTP request. If Loki doesn't respond, the request is destroyed and resolved with an error message.
39 const ok = res.statusCode >= 200 && res.statusCode < 300; Any 2xx is success. Loki returns 204 normally. Resolve the promise with { ok, status, message }.
40 req.on('error', (err) => resolve({ ok: false, ... })); Network errors (DNS failure, connection refused) resolve (not reject) with ok=false. No unhandled promise rejections.
41 req.on('timeout', () => { req.destroy(); resolve(...); }); After 5 seconds, destroy the socket and resolve. Without req.destroy(), the connection hangs open wasting resources.
42 destroy() { this._stopFlushTimer(); this._flushLoki(); } Graceful shutdown: stop the timer (no more scheduled flushes), then flush whatever's in the buffer right now. Call on SIGTERM.
