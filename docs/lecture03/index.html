Grafana Cloud, Loki & Log Shipping

Ship structured logs from your Node.js app to Grafana Cloud Loki, query them with LogQL, and monitor everything through an admin dashboard — all without ELK or heavyweight agents.

🧠 Core Concepts

The building blocks of cloud-based log management and observability.

🔭

Observability

The ability to understand what your app is doing from the outside — by looking at its outputs (logs, metrics, traces). If something breaks at 3 AM, observability means you can figure out why without SSH-ing into the server.

📊

Grafana Cloud

A hosted observability platform. It provides Grafana (dashboards), Loki (logs), Prometheus (metrics), and Tempo (traces) — all managed for you. Free tier is generous enough for small projects.

📋

Grafana Loki

A log aggregation system that indexes only labels, not the full log text. Much cheaper and simpler than Elasticsearch. Query logs with LogQL. Think of it as "Prometheus, but for logs."

🏷️

Log Labels

Key-value pairs attached to a log stream: {app="pipeline", version="3.1.6"}. Loki uses labels for fast lookup — like database indexes. Keep labels low-cardinality (few unique values) for performance.

🚛

Log Transport Modes

How logs get from your app to Loki. Our logger supports 3 modes: local (stdout only), direct (HTTP push to Loki), alloy (stdout for Grafana Alloy agent to tail). Switch at runtime via the admin dashboard.

📡

Loki Push API

A REST endpoint at /loki/api/v1/push that accepts JSON log entries. Our logger batches entries (10 max or every 2 seconds) and pushes them over HTTPS with Basic auth. Each push contains a stream (labels + values).

🖥️

Admin Dashboard

A protected page at /admin showing request metrics, live log stream, and transport controls. Toggle between Local/Direct/Alloy, test the Loki connection, and reset metrics — all from the browser.

📝

Structured Log Metadata

Every log entry includes app, version, level, timestamp, and message. Actions are described in detail: "New item added to the list — 'Widget' assigned id 4 (total items: 4)". Makes debugging in Grafana effortless.

logger.js — Transport Modes & Loki Push

1 const TRANSPORTS = ['local', 'direct', 'alloy']; The three supported transport modes. local = stdout only, direct = push to Loki, alloy = stdout for agent to tail.
2 const APP_NAME = process.env.APP_NAME || require('../package.json').name; App name from env var or package.json. Used as a Loki label. We set APP_NAME=pipelinemaster on the master deployment so logs are separate.
3 const APP_VERSION = require('../public/version.json').version; Reads version from version.json (updated by npm run bump). Every log entry and Loki label includes this — filter logs by release version.
4 this.lokiUrl = process.env.LOKI_URL || ''; Grafana Cloud Loki push URL (e.g., https://logs-prod-012.grafana.net). Stored as a GitHub Secret, injected via ecosystem.config.js.
5 this.lokiUser = process.env.LOKI_USER || ''; Your Grafana Cloud user ID (a number). Used for HTTP Basic Authentication when pushing to Loki.
6 this.lokiToken = process.env.LOKI_TOKEN || ''; Grafana Cloud API token with write scope. A read-only token will return 403. Generate in Grafana Cloud → API Keys.
7 this._lokiBuffer = []; // batch: 10 max or every 2s Logs are buffered in memory before pushing. Flushed when buffer hits 10 entries or every 2 seconds — whichever comes first. Reduces HTTP overhead.
8 if (this.transport === 'direct') this._startFlushTimer(); Only start the 2-second flush timer if transport is "direct". For local/alloy, no timer is needed — logs just go to stdout.
9 this._recentLogs = []; // ring buffer, max 200 The last 200 log entries are kept in memory. The admin dashboard reads these via GET /api/admin/logs — no Loki needed for recent logs.
10 setTransport(mode) { // switch at runtime Change transport mode without restarting. The admin dashboard calls PUT /api/admin/transport which calls this method. Stops/starts the Loki flush timer as needed.

logger.js — The Loki Push Method

11 const ns = new Date(e.timestamp).getTime() + '000000'; Loki needs nanosecond timestamps. JavaScript gives milliseconds — we append 6 zeros to convert. Example: 1712345678123 → 1712345678123000000.
12 stream: { app: APP_NAME, version: APP_VERSION, environment: ... } Labels attached to the log stream. Loki indexes these for fast queries. Filter with {app="pipeline", version="3.1.6"} in LogQL.
13 values: [[ns, JSON.stringify(e)]] Each value is a [timestamp, log_line] pair. The log line is the full JSON entry as a string. Loki stores this as-is — parsed at query time with | json.
14 'Authorization': 'Basic ' + Buffer.from(user + ':' + token).toString('base64') HTTP Basic Auth — LOKI_USER:LOKI_TOKEN base64-encoded. Grafana Cloud requires this. The token must have write scope or you get 403.
15 const ok = res.statusCode >= 200 && res.statusCode < 300; Loki returns 204 (No Content) on success. Any 2xx is OK. A 403 means wrong token scope. A 429 means rate limited.
16 this._pushToLoki(batch).catch(() => {}); // silently drop If Loki push fails, we silently drop the batch. Logs are already on stdout (captured by pm2). The app never crashes because of a logging failure.

app.js — Admin Routes & Metrics

17 app.use((req, res, next) => { // metrics middleware Runs on every request. Records start time, then on res.finish computes duration and feeds it to the metrics collector.
18 if (duration > 1000) log.warn('Slow response detected', { durationMs }); Logs a warning when any response takes more than 1 second. Helps catch performance issues in Grafana — filter with |= "Slow response".
19 res.set('WWW-Authenticate', 'Basic realm="Pipeline Admin"'); Tells the browser to show a login popup. Without this header, the browser gets a 401 but doesn't know to prompt for credentials — we learned this the hard way!
20 app.put('/api/admin/transport', basicAuth, (req, res) => { ... }); Switches log transport mode at runtime. The admin clicks a button, it sends a PUT, and the logger changes from local to direct (or vice versa) instantly.
21 app.post('/api/admin/test-loki', basicAuth, async (req, res) => { ... }); Tests the Loki connection by pushing a single test entry. Returns { ok: true/false, status, message }. Used by the admin dashboard's "Test Connection" button.
