5. Deploy SSH Key — ✅ KEY EXISTS, copy to test server

Why? GitHub Actions needs a way to log into your droplet without a password. SSH keys work like a lock-and-key system: the public key goes on your server (the lock), and the private key goes into GitHub Secrets (the key). When the pipeline runs, it uses the private key to prove its identity to your server.

✅ Key already created: The deploy key pair already exists at ~/.ssh/deploy_key on your Mac. You do NOT need to generate a new one. Just copy it to your new test server.
# Run this from your Mac — copies the key automatically: ssh-copy-id -i ~/.ssh/deploy_key.pub root@NEW_TEST_IP
ssh -i ~/.ssh/deploy_key root@NEW_TEST_IP # If you get a shell prompt, it works! Type 'exit' to disconnect.

6. Add Secrets to GitHub

Why? Your pipeline needs the server IP, username, and SSH key to deploy. You NEVER put these in code. GitHub Secrets encrypts them and makes them available to your workflow as ${{ secrets.NAME }} variables.

Where: GitHub → Your Repo → Settings → Secrets and variables → Actions → New repository secret
# Run this on your Mac to copy the private key: cat ~/.ssh/deploy_key # Copy the ENTIRE output including: # -----BEGIN OPENSSH PRIVATE KEY----- # ... (key content) ... # -----END OPENSSH PRIVATE KEY----- # Paste it as the secret value on GitHub.
